# 專案名稱：線上算命系統

## 專案描述
這是一個提供使用者進行線上算命、占卜與抽籤的平台，讓使用者隨時隨地能尋求指引，並可以儲存結果及進行線上捐獻（香油錢）。

## 1. 專案概述

### 背景與動機
現代人生活壓力大，常面臨諸多抉擇而感到迷惘，需要心靈上的慰藉與指引。傳統宮廟及算命館需要親自前往，對於忙碌的現代人來說較為不便。因此，開發一款線上算命系統，結合傳統民俗信仰與現代科技，讓使用者可以不受時間、空間限制進行線上抽籤與占卜，提供便利的心靈支持。

### 目標用戶
- 對於未來感到迷惘，需要指引方向的年輕族群與上班族。
- 平時有拜拜、抽籤習慣，但沒時間時常造訪實體廟宇的信眾。
- 喜歡嘗試各種算命、占星、占卜以尋求趣味或心理慰藉的民眾。

### 核心價值主張
- **隨時隨地**：24 小時無休，打開網頁即可進行抽籤或占卜。
- **個人化紀錄**：清楚紀錄過去的占卜結果，方便使用者回顧應證。
- **數位信仰**：提供數位化的香客捐獻體驗，讓心意無遠弗屆。

## 2. 功能需求

以下為系統的主要功能清單，包含對應的使用者故事：

### (1) 多元算命與抽籤服務
- **作為** 一名使用者，**我希望** 能夠在線上進行擲筊與抽籤，**以便** 當遇到難題時求取神明指示。
- **作為** 一名使用者，**我希望** 系統能夠提供籤詩的白話文解說，**以便** 我能快速理解籤詩的涵義。

### (2) 儲存與回顧算命結果
- **作為** 一名會員，**我希望** 能夠將每次抽到的籤詩與占卜結果儲存起來，**以便** 日後可以隨時回顧與應證。

### (3) 在線捐獻香油錢
- **作為** 一名信徒，**我希望** 能夠透過線上支付或轉帳（如：模擬第三方金流或支付頁面）捐獻香油錢，**以便** 答謝神恩並支持系統維運。

### (4) 會員註冊與登入系統
- **作為** 一名使用者，**我希望** 能夠註冊專屬帳號並登入系統，**以便** 管理個人資料及專屬的算命紀錄。

### (5) 每日運勢推播
- **作為** 一名會員，**我希望** 登入後能看到個人專屬的每日運勢建議，**以便** 為一天的行程安排提供參考。

### (6) 結果分享
- **作為** 一名使用者，**我希望** 能夠將有趣的占卜結果分享至社群媒體（如 Line、Facebook），**以便** 與親友交流討論。

## 3. 非功能需求

### 技術限制
- **前端與後端框架**：使用 Python 的 Flask 框架搭配 Jinja2 模板引擎進行開發。
- **資料庫**：使用 SQLite 作為關聯式資料庫，便於輕量化部署。
- **架構**：遵循 MVC 架構設計，保持前後端程式碼清晰易讀。

### 效能與安全考量
- **安全性**：確保使用者密碼以雜湊（Hash）方式儲存（例如 bcrypt），防止資料外洩；並防範 SQL Injection 與 XSS。
- **效能**：網站回應時間需保持在合理範圍內，系統需設計成可承受基礎流量並提供流暢的操作體驗。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須具備)
- 基本的系統登入、註冊模組。
- 線上抽籤與擲筊功能，並顯示籤詩內容及基礎解籤。
- 儲存並可查詢個人歷史的算命與抽籤紀錄。

### Should Have (應該具備)
- 模擬捐獻香油錢的功能與感謝介面。
- 管理員後台，可新增或修改籤詩庫內容。

### Nice to Have (有了更好)
- 運勢分享（如一鍵分享至社群媒體）。
- 每日運勢提醒功能。
- 新增更多種類的卜算（例如：塔羅牌）。
